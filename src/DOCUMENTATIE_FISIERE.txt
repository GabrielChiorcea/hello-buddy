📁 MODUL GRAPHQL (/src/graphql)
================================================================================

client.ts
---------
FUNCȚIE: Configurează clientul Apollo pentru comunicarea cu API-ul GraphQL.
CONȚINE:
  - Conexiune HTTP către endpoint-ul GraphQL
  - Link pentru gestionarea erorilor (erori GraphQL și de rețea)
  - Link pentru autentificare (adaugă token Bearer automat)
  - Cache în memorie cu politici pentru entități
EXPORTĂ: apolloClient (instanța configurată)

queries.ts
----------
FUNCȚIE: Definește toate interogările GraphQL pentru citirea datelor.
CONȚINE:
  - Fragmente reutilizabile (ProductFields, UserFields, OrderFields, AddressFields)
  - Query-uri pentru produse (GET_PRODUCTS, GET_PRODUCT_BY_ID, SEARCH_PRODUCTS)
  - Query-uri pentru utilizatori (GET_CURRENT_USER)
  - Query-uri pentru comenzi (GET_USER_ORDERS, GET_ORDER_BY_ID)
  - Query-uri pentru adrese (GET_USER_ADDRESSES)
EXPORTĂ: Toate query-urile și fragmentele

mutations.ts
------------
FUNCȚIE: Definește toate mutațiile GraphQL pentru modificarea datelor.
CONȚINE:
  - Mutații autentificare (LOGIN, SIGNUP, LOGOUT)
  - Mutații profil (UPDATE_PROFILE, REQUEST_PASSWORD_RESET, DELETE_ACCOUNT)
  - Mutații adrese (CREATE_ADDRESS, UPDATE_ADDRESS, DELETE_ADDRESS)
  - Mutații comenzi (CREATE_ORDER, CANCEL_ORDER)
  - Mutații recenzii (ADD_REVIEW)
EXPORTĂ: Toate mutațiile

types.ts
--------
FUNCȚIE: Definește tipurile TypeScript pentru operațiunile GraphQL.
CONȚINE:
  - Tipuri pentru răspunsuri Query (GetProductsResponse, etc.)
  - Tipuri pentru input-uri Mutation (LoginInput, CreateOrderInput, etc.)
  - Tipuri pentru răspunsuri Mutation (AuthResponse, etc.)
  - Tipuri pentru variabile Query și Mutation
EXPORTĂ: Toate interfețele TypeScript

index.ts
--------
FUNCȚIE: Punct central de export pentru modulul GraphQL.
EXPORTĂ: client, queries, mutations, types, hooks

================================================================================
📁 HOOK-URI GRAPHQL (/src/graphql/hooks)
================================================================================

useProducts.ts
--------------
FUNCȚIE: Hook-uri pentru gestionarea produselor.
CONȚINE:
  - useProducts(): Obține toate produsele
  - useProduct(id): Obține un produs după ID
  - useProductsByCategory(category): Filtrează după categorie
  - useSearchProducts(): Căutare produse
  - useCategories(): Lista categoriilor

useAuth.ts
----------
FUNCȚIE: Hook-uri pentru autentificare și gestionarea sesiunii.
CONȚINE:
  - useCurrentUser(): Obține utilizatorul autentificat
  - useLogin(): Autentificare cu email/parolă
  - useSignup(): Înregistrare utilizator nou
  - useLogout(): Deconectare
  - useRequestPasswordReset(): Resetare parolă
  - useDeleteAccount(): Ștergere cont

useProfile.ts
-------------
FUNCȚIE: Hook pentru actualizarea profilului.
CONȚINE:
  - useUpdateProfile(): Actualizează datele utilizatorului

useAddresses.ts
---------------
FUNCȚIE: Hook-uri pentru gestionarea adreselor de livrare.
CONȚINE:
  - useUserAddresses(userId): Lista adreselor utilizatorului
  - useCreateAddress(): Adaugă adresă nouă
  - useUpdateAddress(): Modifică adresă existentă
  - useDeleteAddress(): Șterge adresă
  - useSetDefaultAddress(): Setează adresa implicită

useOrders.ts
------------
FUNCȚIE: Hook-uri pentru gestionarea comenzilor.
CONȚINE:
  - useUserOrders(userId): Istoricul comenzilor
  - useOrder(id): Detalii comandă
  - useCreateOrder(): Plasează comandă nouă
  - useCancelOrder(): Anulează comandă

index.ts
--------
FUNCȚIE: Export central pentru toate hook-urile GraphQL.

📁 SERVICII HTTP (/src/services)
================================================================================

httpClient.ts
-------------
FUNCȚIE: Client HTTP centralizat pentru cereri API cu gestionare JWT.
CONȚINE:
  - Injectare automată a accessToken în header Authorization
  - Refresh automat al token-ului când expiră (răspuns 401)
  - Coadă de cereri în așteptare în timpul refresh-ului
  - Metode helper: get(), post(), put(), patch(), del()
IMPORTANT: Token-urile sunt citite din Redux (memorie), NU din localStorage!
